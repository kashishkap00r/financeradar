# CLAUDE.md

This file is a compatibility guide for Claude Code users.

The canonical repository contributor + handoff documentation now lives in [`AGENTS.md`](./AGENTS.md). Treat that file as the source of truth for project structure, commands, coding rules, testing expectations, PR standards, and handoff format.

## Quick Commands
```bash
python3 aggregator.py
python3 telegram_fetcher.py
GEMINI_API_KEY="..." OPENROUTER_API_KEY="..." python3 ai_ranker.py
GEMINI_API_KEY="..." OPENROUTER_API_KEY="..." python3 wsw_ranker.py
python3 -m unittest discover -s tests
python3 -m http.server 8000
```

## Claude-Specific Working Rules
- Do not hand-edit generated artifacts (`index.html`, files in `static/` generated by scripts).
- For UI changes, edit `templates/app.js` and `templates/style.css`, then regenerate with `python3 aggregator.py`.
- If editing inline frontend templates in `aggregator.py`, use doubled braces (`{{` and `}}`) for literal JS/CSS braces inside f-strings.
- Prefer merging `origin/main` over rebasing when generated files frequently change.

## Handoff
Use the handoff checklist in [`AGENTS.md`](./AGENTS.md). Keep final updates explicit about commands run, tests, generated files touched, and open risks.
